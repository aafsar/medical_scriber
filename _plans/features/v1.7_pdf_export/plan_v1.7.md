# v1.7: PDF Export — Plan & Context

> **Goal:** Add PDF download alongside existing Markdown, producing professional clinical documents.
> **Outcome:** Implemented. PDF export works alongside Markdown with branded header/footer, two-column patient info, teal section headers, and Latin-1 safe text handling.
> **Priority:** 1
> **Depends on:** None

---

## Pre-Conditions

- `build_download_markdown` already works in `note_generator.py`
- Logo exists at `assets/Angy_logo_color.png`
- Streamlit download button pattern established in `app.py`

## Library Choice

**fpdf2** — pure Python, no system dependencies (critical for future Streamlit Cloud deployment in v1.1), simple imperative API, ~1 MB install. Alternatives considered: reportlab (heavier, canvas-based), weasyprint (requires pango/cairo system deps, breaks cloud deployment).

## PDF Layout

- **Page:** Letter (8.5x11"), portrait, 20mm margins
- **Header (every page):** Logo left + "CONSULTATION NOTE" right + DOS + horizontal line
- **Patient Info block:** Two-column layout (Name/DOB/DOS left, Referring Physician/Specialty right), separator line below
- **Clinical sections (12):** Teal bold title with underline, body in regular 10pt, "Not discussed" in italic gray
- **Footer (every page):** "Generated by Angy Voice | AI-assisted — verify all content" + page number
- **Fonts:** Built-in Helvetica only (avoids filesystem issues on Streamlit Cloud)

## Error Handling

- Logo not found → skip logo in header (`Path.exists()` check)
- Logo corrupt → try/except around `pdf.image()`, skip silently
- PDF generation fails → catch in `app.py`, show warning, Markdown still available
- Long text → `multi_cell` + `set_auto_page_break(margin=20)` handles wrapping and pagination
